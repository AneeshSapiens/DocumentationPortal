9. Enhancements

The following are the enhancements implemented in the 7.8.15 release:

New Features

More Information

Optimize DT scaling on Kubernetes

The following properties have been added to the dt-repo, dt-generate, dt-exec, and dt-repair application configuration maps to enhance memory efficiency and optimize CPU utilization:

  • tg.progress.update.seconds

  • tg.execution.result.buffer

  • tg.execution.result.concurrency

  • tg.generate.result.buffer

  • tg.generate.result.concurrency

  • tg.repair.result.buffer

  • tg.repair.result.concurrency

Refer to Decision Manager (DM) Configuration Guide > Customized Configuration Properties> Decision Testing (DT) Advanced Configuration
Change in Flow Association Behavior

Flows can now be associated with only one community at a time. When associating a Flow with a new community, a dissociation request from the current community will be automatically added. Users cannot remove the dissociation request once an association request has been added. Additionally, an association request cannot proceed unless a dissociation request exists for the same Flow.

 Refer to Decision Manager (DM) User Guide > Repository Module > Repository Tasks > Associate/Dissociate Assets

Refer to Current Upgrade Considerations section
Fixing Security Vulnerabilities in DT

Improved security by fixing the following security vulnerability caused by third-party dependencies in DT:

  • Updated tomcat-embed-core from 10.1.34 to 10.1.35

  • Upgraded netty-common from 4.1.114.Final to 4.1.119.Final

  • Upgraded spring-boot from 3.2.11 to 3.3.10

 No impact in any document

Automated SAML Metadata Synchronization

Certificate and SAML metadata synchronization can now be automated by dynamically retrieving metadata from a client-provided Identity Provider (IdP) URL. This enhancement eliminates the need for manual updates when clients modify their SAML metadata, ensuring continuous, secure, and reliable communication between systems

Refer to Decision Manager (DM) Configuration Guide > Authorization and Authentication > Single sign-on (SSO) authentication > Configuring DM for SSO authentication
